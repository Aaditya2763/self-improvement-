# Pull Requests (Code Review)

## Learning Objectives
- Create effective pull requests
- Conduct code reviews
- Manage review feedback
- Maintain code quality

## Topics

### 1. PR Best Practices
- Small, focused PRs
- Clear descriptions
- Single responsibility
- Reference issues

### 2. PR Description Template
```markdown
## Description
Brief explanation of changes

## Type of Change
- [ ] Bug fix
- [ ] New feature
- [ ] Breaking change
- [ ] Documentation

## How Has This Been Tested?
Describe testing approach

## Screenshots (if applicable)
Include relevant images

## Checklist
- [ ] Tests added/updated
- [ ] Documentation updated
- [ ] No breaking changes
```

### 3. Code Review Process
- Review for quality
- Check for tests
- Verify documentation
- Test locally if needed

### 4. Handling Feedback
- Respond to comments
- Make requested changes
- Push commits
- Request re-review

## Guidelines

### PR Title Format
```
[TYPE] Brief description
- [FEAT] Add user authentication
- [FIX] Fix login button bug
- [DOCS] Update README
```

### Do's and Don'ts
✅ Do: Small, focused PRs
✅ Do: Write clear descriptions
✅ Do: Run tests before submitting
✅ Do: Address all feedback
❌ Don't: Submit large PRs
❌ Don't: Skip tests
❌ Don't: Ignore feedback
❌ Don't: Force push after review

## PR Workflow

```
1. Create feature branch
2. Make changes and commit
3. Push to remote
4. Open PR with description
5. Request reviewers
6. Address feedback
7. Get approval
8. Merge and delete branch
```
